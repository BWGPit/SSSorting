Insertion sort

Insertion sort is described as follows, note that arrays are 1-indexed in this pseudocode.
This algorithm's running time is O(n2) . There is a loop invariant which proves the correctness of Insertion sort:
At the start of each iteration, the subarray A[1..j-1] stores the first j-1 elements of A in sorted order. 

INSERTION-SORT(A)
FOR j = 2 TO A.length DO
{
    key = A[j];
    i = j-1;
    WHILE i > 0 && A[i] > key DO
        {
            A[i+1] = A[i];
            i--;
        }
    A[i+1] = key;
}

Selection sort

Selection sort is the worst sorting algorithm out there.
It runs in Θ(n2) as proven by the following equation, which describes each and every time the innermost part of the loop runs. \[\sum_{i=0}^{n-1}n-i = \frac{n(n-1)}{2} \in \Theta (n^2)\] The pseudocode is almost not needed as the algorithm is extremely simple and designed in a brute-force fashion, although it is probably the most natural way of solving this problem, albeit inefficiently. 
SELECTION-SORT(A)
FOR i = 1 TO A.length-1 DO
{
    min = i;
    FOR j = i+1 TO A.length DO
    {
        IF A[j] < A[min] THEN min = j;
    }
    SWAP(A[i], A[min]); // SWAP is defined as a usual procedure for swapping variables
}

Merge sort

Merge sort is an example of divide-and-conquer algorithms (in Latin, "divide et impera").
Its running time is Θ(n log n). The pseudocode is described as follows. 
MERGE-SORT(A, p, r)
IF p < r THEN
{
    q = ⌊ (p+r)/2 ⌋;
    MERGE-SORT(A, p, q);
    MERGE-SORT(A, q+1, r);
    MERGE(A, p, q, r);
}
MERGE(A, p, q, r)
n1 = q-p+1;
n2 = r-q;
L[1..n1+1] = new Array;
R[1..n2+1] = new Array;
FOR i = 1 TO n1 DO L[i] = A[p+i-1];
FOR j = 1 TO n2 DO R[j] = A[q+j];
L[n1+1] = ∞;
R[n2+1] = ∞;
i = 1;
j = 1;
FOR k = p TO r DO
{
    IF L[i] ≤ R[j] THEN
    {
        A[k] = L[i];
        i++;
    }
    ELSE
    {
        A[k] = R[j];
        j++;
    }
}

Quick sort

Quick sort is probably the best sorting algorithm described here, as it is generally the quickest among them and works in place, while Merge sort has an additional space cost.
Although Quick sort's worst-case time complexity is Θ(n2), its average-case running time is Θ(n log n) and it has small hidden constants. The pseudocode is described as follows. 
QUICK-SORT(A, p, r)
IF p < r THEN
{
    q = PARTITION(A, p, r);
    QUICK-SORT(A, p, q-1);
    QUICK-SORT(A, q+1, R);
}
PARTITION(A, p, r)
x = A[r];
i = p-1;
FOR j = p TO r-1 DO
{
    IF A[j] ≤ x THEN
    {
        i++;
        SWAP(A[i], A[j]); // SWAP is defined as a usual procedure for swapping variables
    }
}
SWAP(A[i+1], A[r]);
RETURN i+1;

Heap sort

This sorting algorithm introduces an interesting data structure which is Heap. By definition, it is a nearly complete binary tree where all leaves are in the left side. This makes it easy to implicitely represent it as an array where each element represents a node and it is possible to jump from a node to its children and to its parent with the following procedures, which take the index of a node as input and return the index of the desired node. Once again, note that here arrays are assumed to be 1-indexed. 
PARENT(i)
RETURN ⌊ i/2 ⌋;
LEFT(i)
RETURN ⌊ 2i ⌋;
RIGHT(i)
RETURN ⌊ 2i+1 ⌋;

Heap sort makes use of a Max-heap to sort the array given in input, which is implicitely converted into said data structure which has the following property: each node's value is greater than, or equal to, that of its children. To maintain this property, the following procedure is used, which runs in O(log n), assuming that the node upon which is called is the root of a Max-heap. 
MAX-HEAPIFY(A, i)
l = LEFT(i);
r = RIGHT(i);
largest = NIL;
IF l ≤ A.heap-size ∧ A[l] > A[i] THEN
{
    largest = l;
}
ELSE
{
    largest = i;
}
IF r ≤ A.heap-size ∧ A[r] > A[largest] THEN
{
    largest = r;
}
IF largest ≠ i THEN
{
    SWAP(A[i], A[largest]); // SWAP is defined as a usual procedure for swapping variables
    MAX-HEAPIFY(A, largest); // Recursively repeat this action upwards towards the tree root
}
The heap-size property is added to the array when the Heap is built. The building process costs O(n). 
BUILD-MAX-HEAP(A)
A.heap-size = A.length;
FOR i = ⌊ A.length/2 ⌋ DOWNTO 1 DO
{
    MAX-HEAPIFY(A, i);
}
With these procedures, the Heap sort algorithm is simply described as follows. Its time complexity is O(n log n). 
HEAP-SORT(A)
BUILD-MAX-HEAP(A);
FOR i = A.length DOWNTO 2 DO
{
    SWAP(A[1], A[i]);
    A.heap-size -= 1;
    MAX-HEAPIFY(A, 1);
}
